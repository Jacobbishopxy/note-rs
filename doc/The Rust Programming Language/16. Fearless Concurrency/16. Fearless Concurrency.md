# 无惧并发

安全和效率处理并发编程是 Rust 的另一个主要目标。随着计算机越来越多的处理器*并发编程*--程序中不同部分独立运行，以及*并行编程*--不同部分在同一时间执行，它们变得越来越重要。由于历史原因，这类编程在上下文中很困难也容易出错：Rust 希望能改变这个。

...

很多语言提供的并发问题处理方案是很有特色的。例如 Erlang 有优雅的消息传递并发功能，但是只有模糊的线程间共享状态方法。对于高级语言，只实现困难解决方案的子集是一个合理的策略，因为高级语言承诺的价值源自于牺牲一些控制来获得抽象。然而底层语言期望提供在任何给定情况下有着最高性能并且对硬件更少的抽象。因此 Rust 对适合你的情况和需求，提供了多种工具对问题建模。

以下是本章覆盖的内容：

- 如何创建多线程来同时运行多段代码
- *消息传递*并发，其中通道用于在线程间传递消息
- *共享状态*并发，其中多个线程可以访问同一片数据
- `Sync`与`Send`特性，扩展 Rust 的并发保障到了用户定义的类型以及标准库提供的类型
